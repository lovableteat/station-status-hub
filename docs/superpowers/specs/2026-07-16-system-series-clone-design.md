# 機台連號複製設計

## 目標

從 L10 測試追蹤表的單一來源機台，批次建立下一段連號機台名稱，例如由 `Golden3` 建立 `Golden4` 到 `Golden8`，並複製可安全沿用的機台與測試進度資料。

## 操作流程

1. 在來源機台右側 `...` 選單點選「複製為連號機台」。
2. 對話框自動拆出名稱前綴與下一個號碼，預設建立 5 台並顯示完整名稱預覽。
3. 使用者可修改前綴、起始號碼與數量，單次最多 100 台。
4. 確認後先建立所有機台，再複製測項進度與儀表板排除設定；完成後重新載入目前專案。

## 資料規則

- 複製：專案、流程版本、工程師、型號、團隊、機櫃、系統版本、CUDA、BOM、機台狀態、目前站點、總進度、各測項狀態、百分比、負責人、備註與儀表板排除設定。
- 不複製：序號、MAC/BMC 位址、問題單、附件、稽核歷史、實際開始／完成時間與實際計時。
- 名稱保留來源尾碼的補零格式，例如 `System003` 的下一台是 `System004`。
- 同一專案出現重複名稱時，整批機台建立請求失敗，不寫入部分名稱。
- 若後續進度複製失敗，立即呼叫既有 `delete_test_system` 清除本次已建立機台。

## 介面

- 新增獨立 `SystemCloneDialog`，不擴大既有新增機台表單。
- 對話框提供來源摘要、三個數值設定、名稱預覽與資料範圍說明。
- 送出期間停用所有欄位，防止重複建立。

## 驗證

- 單元測試驗證尾碼解析、補零、名稱預覽與數量限制。
- 原始碼測試驗證每列操作已接上複製對話框，並驗證服務會複製進度且在失敗時回滾。
- 執行完整 Node 測試、ESLint、production build 與正式部署資源檢查。
