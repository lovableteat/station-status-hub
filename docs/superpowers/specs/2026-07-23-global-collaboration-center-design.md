# 全站協作中心設計

## 目標

將目前彼此分離且資訊不足的通知彈窗與在線用戶彈窗，整合成一個可操作的全站協作中心，並讓後台管理能發送全體或指定帳號公告、檢視在線位置及發送紀錄。

## 使用者介面

- Header 鈴鐺開啟「通知」分頁，Header 在線按鈕開啟同一面板的「在線成員」分頁。
- 通知分頁提供未讀數、單筆已讀、全部已讀、重新載入、空狀態及錯誤狀態。
- 在線分頁顯示包含自己的全站在線人數、姓名、角色、目前頁面及編輯狀態。
- 手機改用近全螢幕面板，桌面使用右上側浮層，不再產生兩套重疊彈窗。

## 後台連結

- 後台管理新增「通知與在線」分頁。
- 管理員可選擇全體啟用帳號或指定帳號，填寫標題與內容後發送。
- 發送由資料庫 RPC 一次完成，避免瀏覽器逐筆新增造成重複或漏發。
- 發送紀錄由既有 `user_notifications` 依批次識別彙整，不複製通知內容到第二張表。

## 資料與相容性

- 延用既有 `user_notifications`，保留 mention、問題回覆與既有通知資料。
- 管理公告使用 `notification_type = admin_announcement`，批次與導覽資訊放在 metadata。
- Presence 延用 Supabase Realtime channel，但回傳全部在線成員、其他在線人數與連線狀態。
- 不刪除任何正式資料，也不更名既有欄位。
